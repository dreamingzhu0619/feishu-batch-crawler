# 批量创建爬虫任务 - 使用指南

## 工具总览

| 工具 | 说明 | 端口 |
|------|------|------|
| `export_companies.py` | 从 MongoDB 导出公司数据为 CSV | 命令行工具 |
| `batch_creator.py` | Web 端批量创建爬虫任务（含天眼查补全、去重、Generate） | 8899 |
| `batch_import.py` | 自动化导入 + 实时看板 | 8888 |

## 典型工作流

```
MongoDB 数据 ──→ export_companies.py ──→ CSV 文件 ──→ batch_creator.py Web 上传 ──→ 创建任务 ──→ Generate 爬虫
```

## 一、环境部署

### 1. 在服务器上执行一键部署

```bash
bash setup.sh
```

此脚本会安装系统依赖、创建 Python 虚拟环境、安装 flask/requests/httpx/fastapi/uvicorn 等包。

### 2. 启动服务

```bash
bash start.sh
```

启动后可访问：
- 批量创建 Web 端：`http://<服务器IP>:8899`
- 导入看板：`http://<服务器IP>:8888/dashboard`

### 3. 停止服务

```bash
bash stop.sh
```

## 二、导出公司数据（export_companies.py）

从 MongoDB `company_one` 库中读取 mokahr / dayi / beisen / feishu 四个集合的职位数据，按公司名去重，自动反推招聘列表页 URL，输出为 CSV 文件。

### 前置条件

- 安装 pymongo：`pip install pymongo`
- 能访问 MongoDB 服务器 `192.168.253.10:27017`

### 基本用法

```bash
# 导出全部 4 个平台（默认输出 companies_export.csv）
python export_companies.py

# 仅导出飞书平台
python export_companies.py --platform feishu

# 导出多个平台
python export_companies.py --platform mokahr dayi

# 指定输出文件名
python export_companies.py --output result.csv
```

### 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--platform` | 导出的平台，可选 `mokahr` `dayi` `beisen` `feishu` `all` | `all` |
| `--output` | 输出 CSV 文件路径 | `companies_export.csv` |

### 输出格式

CSV 包含三列：

```csv
company_name,url,channel
北京月之暗面科技有限公司,https://app.mokahr.com/social-recruitment/moonshot/148506#/,mokahr
天翼云科技有限公司,https://wecruit.hotjob.cn/SU62b2ae672f9d24458d72f9cc/pb/index.html#/social,dayi
追觅科技（苏州）有限公司,https://dreame.zhiye.com/social/jobs,beisen
WeWork中国,https://jobs.wework.cn/application,feishu
```

| 字段 | 说明 |
|------|------|
| `company_name` | 公司全称（来自 MongoDB） |
| `url` | 招聘列表页 URL（由职位详情 URL 反推） |
| `channel` | 平台来源标识：mokahr / dayi / beisen / feishu |

### URL 反推规则

各平台从存储的职位详情 URL 反推到列表页 URL 的逻辑如下：

**mokahr** — 截取 `#/job/` 前面的部分，拼接 `#/`
```
职位页: https://app.mokahr.com/social-recruitment/deeproute/143885#/job/363189b3-...
列表页: https://app.mokahr.com/social-recruitment/deeproute/143885#/
```

**dayi** — 去掉路径中的 `/wecruit` 前缀，截到 `/pb` 前，拼接 `/pb/index.html#/social`
```
职位页: https://wecruit.hotjob.cn/wecruit/SU62b2ae67.../pb/posDetail.html?postId=...
列表页: https://wecruit.hotjob.cn/SU62b2ae67.../pb/index.html#/social
```

**beisen** — 取域名，拼接 `/social/jobs`（自动处理异常 URL 中的重复域名）
```
职位页: https://hankun.zhiye.com/social/detail?jobAdId=...
列表页: https://hankun.zhiye.com/social/jobs
```

**feishu** — 截取 `/position` 前面的部分
```
职位页: https://ponyai.jobs.feishu.cn/ponyai/position/7563.../detail
列表页: https://ponyai.jobs.feishu.cn/ponyai

职位页: https://jobs.wework.cn/application/position/...
列表页: https://jobs.wework.cn/application
```

## 三、Web 端批量创建任务（batch_creator.py）

浏览器访问 `http://<服务器IP>:8899`，有两种数据加载方式。

### 方式 A：上传 CSV 文件

1. 在页面顶部切换到 **「上传 CSV」** 标签页
2. 点击上传区域，选择 `export_companies.py` 导出的 CSV 文件
3. 上传后自动解析并显示预览表格

> 也可以点击「下载 CSV 模板」获取格式参考，手动编辑后上传。

### 方式 B：加载内置 MD 文件

1. 保持在 **「内置 MD 文件」** 标签页
2. 点击 **「加载并预览」**，自动解析 `url输出结果.md`

### 执行创建

1. 预览数据确认无误后，点击 **「开始创建」**
2. 「开始创建」按钮旁有 **「创建完成后自动 Generate」** 勾选框（默认勾选）
3. 系统逐条处理每家公司：
   - 先检查 URL 是否已存在（去重）
   - 调用天眼查 API 补全公司简称和信用代码
   - 调用平台 API 创建爬虫任务
4. 实时显示进度条和统计数据
5. 全部创建完成后，若勾选了自动 Generate，系统自动对所有创建成功的公司执行 Generate，无需手动操作

### 处理结果状态

| 状态 | 含义 |
|------|------|
| 创建成功 | 天眼查查到简称，创建成功 |
| 降级创建 | 天眼查未查到，使用原始公司名创建 |
| 已跳过 | URL 已存在，跳过重复项 |
| 创建失败 | 创建出错，可编辑后重跑 |

### 失败重跑

对于创建失败的条目：
1. 点击该行的 **「编辑」** 按钮
2. 修改简称或信用代码
3. 点击 **「重跑」**

### Generate 爬虫配置

**自动模式（推荐）：** 保持「创建完成后自动 Generate」勾选，创建完成后自动开始 Generate 全部成功的公司，整个流程一键完成。

**手动模式：** 取消勾选后，创建完成后页面底部会出现 Generate 区域，可手动操作：
1. 勾选需要 Generate 的公司（或点击表头全选）
2. 点击 **「Generate 选中」** 或 **「一键全部 Generate」**
3. 系统串行调用 Generate API，日志实时滚动显示
4. 如需中断，点击 **「停止 Generate」**

## 四、完整操作示例

```bash
# 1. 本地导出全部平台数据
python export_companies.py
# 输出: 共导出 186 家公司 → companies_export.csv

# 2. 在服务器上启动 Web 服务
bash start.sh

# 3. 浏览器打开 http://<服务器IP>:8899
```

如果只需要某个平台的数据：

```bash

```

## 六、常见问题

**Q: export_companies.py 报连接失败**
A: 确认能访问 MongoDB `192.168.253.10:27017`，检查网络和防火墙。脚本默认 5 秒超时。

**Q: CSV 上传后提示解析为空**
A: 检查 CSV 第一行表头必须包含 `company_name` 字段。建议用 UTF-8 编码保存。

**Q: 某公司创建失败提示 URL 格式错误**
A: 在 Web 端点击「编辑」修正 URL 后重跑，或在 CSV 中修正后重新上传。

**Q: Generate 超时或失败**
A: Generate 单个公司可能耗时较长（超过 10 分钟属正常）。失败后可勾选该公司重新 Generate。
